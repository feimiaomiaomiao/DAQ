******************************************************************************
程序ADC数据存储：
ADC1多通道（通道1，2，16，17）采集数据，通过DMA1转存到数组ad_result[4]中
ad_result[0]   存放内部通道17数据（MCU内部通道，内部参照电压）
ad_result[1]   存放通道1数据，此程序连接的是40K-无穷大的，量程5KG的压力传感器（压力越大，阻值越小）
ad_result[2]   存放通道16数据（MCU内部通道，温度传感器）
ad_result[3]   存放通道2数据，此程序连接的滑动变阻器2

ADC内部引脚与温度转换，资料参考：
参考手册11.3.3（中157p），参考手册11.10（中169p），数据手册5.3.18（中51p）
注意事项：
DMA1通道1配置，能且只能通道1参考手册10.3.7（中147p）
(DMA1的通道和ADC1的通道不是一个概念，不要混淆)
******************************************************************************

******************************************************************************
关于优化，在ADC引脚加滤波电容的作用：
	滤除高频噪声：为高频干扰信号提供到地的低阻抗路径
	稳定ADC读数：减少电压波动，提供更稳定的采样值
	防止ADC引脚振荡：ADC输入引脚有容性负载，可能产生振荡
******************************************************************************
	
	
******************************************************************************
ADC+DMA初始化步骤：
1，打开ADC1和DMA1和对应GPIO（PA1,PA2）引脚的时钟（数据手册第3章（中17p））
2，配置对应GPIO引脚为模拟输入（ADC1用的通道1（PA1），通道2（PA2）
   内部通道16（温度传感器）内部通道17（Vrefint），所以打开PA1,PA2即可
3，配置DMA1，目的是实现ADC1->DR寄存器里面的数据搬到RAM【ad1_result】数组中
4，打开DMA1使能
5，配置ADC1多通道单次模式（必须为单次模式，因为要刷新OLED，连续采集会破坏OLED时序，导致显示错误）

6，配置ADC1采样周期
   其中采样列表，通道17在第一位，获取了基准电压后，修正后续的其他通道数据
   采样列表：1-ADC1通道17，2-ADC1通道1，3-ADC1通道16，4-ADC1通道2
7，使能ADC1的DMA功能
8，使能ADC1内部通道
9，使能ADC1
10，ADC1校准
11，启动ADC1转换

之后CPU不用做任何数据搬运操作，数据会自动存在ad1_result数组中对应的位置

******************************************************************************
为什么要用DMA，而不是读取ADC中断标志：
扫描模式时，会持续转换把列表内的通道数据存在ADC->DR寄存器里面，
比方有5个ADC数据采集，只有列表最后才会产生EOC转换结束标志位。
这样会导致ADC->DR寄存器里面的数据会被覆盖，即前面4个数据如果没有即使取走，会被覆盖
但是DMA不同，每次转换一个通道结束后，会产生一个DMA请求，这样可以调用DMA去转存数据
